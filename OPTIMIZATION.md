# Activiti 工作流维护工具 - 优化说明

## 界面逻辑优化

### 1. 操作流程重新设计

优化后的操作流程更加清晰直观：

```
启动程序
    ↓
连接数据库界面（默认显示）
    ↓
连接成功
    ↓
Activiti工作流维护工具界面（自动跳转）
    ├─ 运行中实例（默认标签页）
    └─ 已结束实例
    ↓
点击"详情"
    ↓
流程实例详情
    ├─ 基本信息
    ├─ 当前任务 → 设置审批人 / 退回任务
    ├─ 流程变量 → 添加/修改/删除变量
    └─ 历史任务 → 退回到此节点
```

### 2. 主要改进

#### 2.1 Activiti工作流维护工具
- **运行中/已结束标签页**：清晰区分流程实例状态
- **状态标签显示**：使用颜色标签直观显示运行状态
- **改进的列表展示**：显示发起人信息而不是用户ID

#### 2.2 流程实例详情
- **查看流程图**：直接查看流程定义XML
- **修改流程图**：仅限运行中的实例进行修改
- **退回任务节点**：从历史任务列表选择目标节点
- **设置审批人**：支持单个审批人和多审批人（IdentityLink）

#### 2.3 用户体验优化
- **自动跳转**：连接成功后自动跳转到流程实例页面
- **断开连接**：在侧边栏显示连接状态和断开按钮
- **清晰的导航**：简化的侧边栏导航

### 4. 技术实现

#### 后端改进
- 支持 `status` 参数区分运行中/已结束实例
- 添加 `isFinished` 字段标识流程状态
- 优化数据库查询性能
- 支持瀚高数据库

#### 前端改进
- 标签页切换逻辑
- 状态标签样式
- 模态框优化
- 响应式布局

#### API 端点
```
GET  /api/instances?status=running|finished  # 获取流程实例
GET  /api/finished-instances                 # 获取已结束的流程实例
GET  /api/instances/:id                      # 获取实例详情
PUT  /api/definitions/:id/xml                # 修改流程图
POST /api/instances/:id/jump-to-task         # 退回任务
POST /api/tasks/:taskId/set-assignee         # 设置审批人
POST /api/tasks/:taskId/add-identitylink     # 添加IdentityLink
DELETE /api/tasks/:taskId/identitylink       # 删除IdentityLink
```

### 6. 常见操作示例

#### 查看已结束的流程实例
1. 点击"已结束"标签页
2. 浏览或搜索已结束的流程
3. 点击"详情"查看完整信息

#### 退回任务节点
1. 打开流程实例详情
2. 滚动到"历史任务"部分
3. 点击目标任务的"退回此处"按钮
4. 确认退回操作

#### 设置审批人
1. 在"当前任务"部分点击"设置审批人"
2. 输入用户ID或添加候选人
3. 点击"保存审批人"

#### 修改流程图
1. 打开流程实例详情
2. 点击"修改流程图"按钮
3. 编辑XML内容
4. 点击"保存修改"

### 7. 注意事项

1. **数据安全**
   - 修改流程图前请备份数据
   - 退回任务会重置流程状态
   - 删除操作不可撤销

2. **性能考虑**
   - 大数据量时请使用搜索功能
   - 分页查看避免一次性加载过多数据

3. **瀚高数据库**
   - 默认端口：5866
   - 与 PostgreSQL 语法兼容
   - 连接配置与其他数据库相同
