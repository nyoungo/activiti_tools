# Activiti 工作流维护工具 - 功能说明

## 一、部署和启动

### 1.1 开发环境运行

```bash
npm install
npm run dev
```

访问 http://localhost:34567

### 1.2 Windows 打包

```bash
npm run build:win
```

打包后的文件在 `build/activiti-tools.exe`

## 二、支持的数据库

- MySQL (默认端口 3306)
- PostgreSQL (默认端口 5432)
- 瀚高数据库 (默认端口 5866)

## 三、主要功能

### 3.1 数据库连接管理

1. 填写数据库连接信息
2. 点击"测试连接"验证连接
3. 点击"保存配置"保存到本地
4. 点击"连接"进入系统

### 3.2 Activiti工作流维护工具

#### 功能说明
- 查看所有运行中的流程实例
- 支持按流程名称、Key、业务Key搜索
- 分页显示，默认每页10条

#### 用户信息显示
- 发起人显示：真实姓名 > 用户名 > 用户ID
- 处理人显示：真实姓名 > 用户名 > 用户ID

### 3.3 流程实例详情

#### 基本信息
- 显示流程名称、Key、业务Key等
- 发起人显示用户真实姓名

#### 当前任务
- 显示当前任务列表
- 支持设置审批人功能
- 支持设置候选人/多审批人

#### 流程变量管理
- 查看、添加、修改、删除流程变量
- 支持 String、Long、Double 类型

#### 历史任务
- 显示历史任务列表
- 支持跳转到历史节点
- 处理人信息显示

### 3.4 流程定义管理

#### 功能说明
- 查看所有流程定义
- 按 Key 和 Version 排序

#### XML 操作
- 查看 XML：只读模式
- 修改 XML：可编辑模式，点击"保存修改"保存

### 3.5 设置审批人功能

#### 单个审批人（Assignee）
1. 点击任务的"设置审批人"按钮
2. 在"设置审批人"输入框输入用户ID
3. 点击"保存审批人"

#### 多审批人（IdentityLink）
1. 点击任务的"设置审批人"按钮
2. 在"添加候选人"输入框输入用户ID
3. 选择类型：候选人 或 审批人
4. 点击"添加"
5. 可在下方列表查看和管理已添加的候选人/审批人

#### 删除候选人/审批人
- 在 IdentityLink 列表中点击"删除"按钮

### 3.6 历史任务跳转

#### 功能说明
将流程实例跳转到指定的历史节点，重置流程状态。

#### 使用方法
1. 打开流程实例详情
2. 找到历史任务列表
3. 点击目标任务的"跳转到此处"按钮
4. 确认操作

#### 注意事项
- 跳转会删除当前所有任务
- 跳转会创建新的任务到目标节点
- 请谨慎操作

## 四、技术架构

### 后端 (src/server.js)
- Express.js Web 服务
- 支持 MySQL、PostgreSQL、瀚高数据库
- RESTful API 设计

### 前端
- 原生 HTML/CSS/JavaScript
- 无需构建工具
- 响应式设计

### 打包
- 使用 pkg 进行打包
- 单文件可执行
- 支持 Windows/macOS/Linux

## 五、数据库要求

### Activiti 表结构
- ACT_RU_* 表：运行时数据
- ACT_HI_* 表：历史数据
- ACT_RE_* 表：流程定义数据
- ACT_GE_* 表：通用数据

### 用户表
- sys_user 表（需要包含 id、username、realname 字段）
- 用于关联显示用户信息

## 六、常见问题

### 1. 打包后静态文件无法加载
检查控制台输出，查看静态文件路径。确保 pkg.assets 配置正确。

### 2. 分页按钮不显示
确保数据库中有足够的流程实例数据（超过 pageSize 设置的值）

### 3. 流程定义页面不显示
确保已连接到数据库，并且数据库中有流程定义数据

### 4. 用户信息显示为ID
检查 sys_user 表是否存在，或用户ID是否在 sys_user 表中有对应记录

### 5. 瀚高数据库连接失败
- 检查端口是否正确（默认 5866）
- 检查数据库驱动是否正确安装
- 验证数据库服务是否正常运行
